关于本站

　　Shaun 个人空间，主要用来记录 Shaun 个人之所见所学所想所得， 以证明 Shaun 存在过，经历过。

　　真正有写一些东西的想法的时候还是在大三期间 （虽然在大一就给人写过一篇关于思考的文章， 19 年回头去看虽然有点小孩子气， 但其中关于思考的部分 Shaun 现在还是认可的 _(:3ゝ∠)_）， 因为那个时候才算是编程真正入门了， 知道如何正确的学习编程，如何利用网络去解决 80% 以上的问题， 如何用编程去实现一些有趣的东西。

　　在利用网络去解决问题的时候，Shaun 发现网络的东西确实良莠不齐， 有些东西比较混乱，甚至是错的，信噪比实在是太低了， 这样查找起来确实很浪费时间，于是 Shaun 就想将这些信息进行归纳总结， 顺便加入一些自己一些微不足道的想法，于是就有了初步的写东西的想法， 刚开始也就是记录自己的实践过程和出现的问题及解决方案， 虽然现在还是 ╮(╯▽╰)╭，一般以纯代码的形式展示，后面发现这样不好， 这样就真的只有自己能看懂了（或许过段时间连自己都看不懂了 o(╯□╰)o）， 后面逐渐改进，慢慢形成了现在这样的风格。

　　最初的时候其实在某博客站点上写的，后面发现这样很不自由， 好像自己写的东西不是自己的一样，后面发现 GitHub Pages 可以当成一个站点， 而且可以完全控制自己站点的样式，基本上是想怎样就怎样了， 而且不需要去管理服务器，不用去操心服务器的事，甚至域名都不用去管， 这基本上是完美符合 Shaun 的需求了，于是在读研期间利用 Hexo 搭了一个， 遂有此站。

关于评论系统

　　2024.11.20 更新：评论系统更改为 giscus。

　　由于 GitHub 本身的权限的问题，Gitalk 的 OAuth 登录问题存在安全隐患， 不推荐直接在评论系统上直接使用 Github 上进行登录， 具体安全隐患问题可参考 Gitment 的安全性争议 、OAuth 授权的工作原理是怎样的？足够安全吗？ 。 所以推荐需要评论时最好直接点击 Gitalk 评论系统左上角的评论数， 将会自动跳转到对应 issue，这样直接在 issue 下发言即可， 可以不需要在 Gitalk 上登录。而不想使用 utterances 的原因是因为其界面不是很好看 ╮(╯﹏╰）╭， 而且有些功能不是很完善。
